External Use Of The Unified Model

1. Introduction

The atmosphere and ocean modelling code used by the Meteorological Office is freely available for use by other research groups and meteorological centres. The code exists within a single software system known as the Unified Model and covers both data assimilation and prediction for global or regional domains. It is the code used by the Meteorological Office in carrying out its operational numerical weather forecasting activities. It is also used by the Hadley Centre for Climate Prediction and Research.

2. Unified Model Software

The code is written in FORTRAN 77 and ANSI C and runs under the Bourne shell of the UNIX operating system. The Unified Model is normally run on a Cray C90 computer and the code is designed to use the automatic multitasking and vectorization facilities available on Cray computers. A workstation version of the code is also available. This runs on computer systems using the IEEE standard for representing floating point numbers.

The dynamics routines and physical parametrizations are written in the plug-compatible form agreed for the international exchange of meteorological subroutines.

The main model components are:

  • User Interface

    An X-Windows application for setting up model integrations. It comprises over 200 separate windows and may be customised by the user.

  • Reconfiguration

    A generalised interpolation package used to convert model data files to new resolutions and areas.

  • Data Assimilation

    Analysis correction scheme using repeated insertion of observations.

  • Atmosphere Model

    Grid point split-explicit dynamics and physical parametrizations.

  • Ocean Model

    Based on the code of Bryan and Cox developed at GFDL.

  • Atmosphere-Ocean Coupling

    Software to run atmosphere and ocean models in coupled mode, including a dynamic sea-ice model.

  • Diagnostics and Output

    Internal model package to output a range of diagnosed and derived quantities over arbitrary time periods, sub-areas and levels.

3. Access and Availability

New releases of the model are made available once they have been accepted by internal users. The data and software are available on tape in the following formats:

  • QIC 150 format on a DL6150 or DL6250 1/4" cartridge
  • DDS format on a 60m or 90m DAT cartridge.

Both the Cray compatible and workstation versions of the code are available and include the following features:

  • Conversion software to pass output to workstations (Cray version only).
  • On-line documentation covering both technical and scientific aspects of the model in Wordperfect 6.0 and Postscript formats. A Mosaic based hypertext browser is also available linking source code viewing and documentation.
  • A small archive of model analyses and all ancillary files needed to run the system.
  • A facility to run the atmosphere model from ECMWF GRIB coded data.

4. Future Developments

It is planned to provide information on software upgrades via an Internet WWW server connected to the Meteorological Office. This service will be introduced during 1995.

A message passing version of the code is under development using PVM and MPI. This will allow the code to be run on distributed memory parallel computer systems or on networks of workstations. The messaging passing version of the atmosphere component will be available in the second half of 1995.

5. Contacts

Two full time members of staff support external use of the model. They provide initial help and advice on installation procedures. If you have an agreed joint research project with the Meteorological Office, you may qualify for further free support. Support may also be available on a repayment basis.

Please note that a small fee will be charged to cover administration costs.
